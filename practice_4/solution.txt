1. Как загрузить готовый файл с компьютера в репозиторий?
Можно двумя способами:

Через сайт GitHub: Add file → Upload files → выбрать файл → Commit changes.

Через Git: добавить файл в папку проекта → git add . → git commit -m "сообщение" → git push.

2. Что делать, если преподаватель не видит репозиторий студента?
Проверить:

добавлен ли преподаватель в Collaborators,

принят ли им инвайт,

правильность ссылки на репозиторий,

не является ли репозиторий приватным без доступа.
При необходимости заново выслать приглашение или сделать репозиторий публичным.

3. Какой тип репозитория рекомендуется выбрать и почему?
Обычно Private (приватный), чтобы:

код был доступен только студенту и преподавателю,

избежать копирования другими,

сохранить академическую честность.
(Если доступ нужен всем — можно Public.)

4. Какой элемент интерфейса используется для отправки Pull Request?
Кнопка New Pull Request (или Compare & pull request) на странице репозитория/ветки.

5. Что произойдет, если файл случайно удалён с сервера GitHub?
Файл можно восстановить:

из предыдущих коммитов (через историю),

повторно загрузив и запушив его.
Git хранит историю, поэтому данные обычно не теряются окончательно.
