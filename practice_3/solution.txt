1. Какие шаги нужно предпринять студенту, чтобы преподаватель получил доступ к репозиторию?
Нужно добавить преподавателя в репозиторий через "Settings → Collaborators (или Collaborators and teams)", указав его GitHub-ник или email. После этого преподаватель принимает приглашение и получает доступ.

2. Чем грозит отсутствие добавления преподавателя в Collaborators?
Преподаватель не сможет:

просматривать код,
проверять коммиты и ветки,
оставлять комментарии,
принимать Pull Request.
Итог — работа может быть не проверена или не засчитана.

3. Для чего создавать отдельные ветки для каждого задания?
Чтобы:
изолировать изменения по каждому заданию,
не ломать основную ветку (main/master),
упростить проверку и ревью,
вести чистую историю коммитов,
параллельно работать над несколькими задачами.

4. Какие элементы обязательно включаются в заголовок и описание Pull Request?
В заголовке:

номер/название задания,
краткое описание изменений.

В описании:

что сделано,
какие файлы/функции изменены,
инструкции по проверке (если нужно),
при необходимости ссылки на задание или комментарии.

5. Как отправить готовую работу преподавателю?
Запушить ветку в GitHub → создать "Pull Request" в основную ветку (main/master) → назначить преподавателя reviewer’ом или уведомить его → дождаться проверки.
